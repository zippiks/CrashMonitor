# CrashMonitor
CrashMonitor - учебный проект, помогающий отследить аварийное завершение программы и сохрянить дамп памяти для дальнейшего анализа

Составляющие:
Agent - запускает программу, следит за ее работой, записывает лог и создает дамп памяти в случае аварийного завершения
CrashApp - тестовое приложение, которое специально падает через 5 секунд после запуска(А так же есть возможность закончить работу штатно - для этого необходимо закомментировать следующие строки в CrashApp.cpp:    //int* ptr = nullptr;    //*ptr = 67;)
CrashReports - папка для хранения дампов памяти
Agent.log - журнал работы агента

Для запуска требуется запустить сам agent. В случае аварийного прекращения работы агент предложит перезапустить приложение или закрыть его.

Принцип работы:
1. Агент запускает тестовое приложение
2. Ждет событие отладчика
3. Если приложение завершается с ошибкой, определяется код исключения
4. Создается файл дампа памяти .dmp, так же вся информация записывается в agent.log
5. При необходимости есть возможость перезапустить приложение
